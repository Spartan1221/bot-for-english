Действуй как Senior Python разработчик. Твоя задача — написать Telegram-бота на Python с использованием библиотеки `aiogram` (версия 3.x). 

## Функционал бота:
Бот принимает от пользователя слово или фразу на английском языке и возвращает ответ в строго отформатированном виде, удобном для выделения и копирования. 
Ответ должен содержать:
1. Изначальное слово и пример его использования.
2. Перевод на русский язык и определение (значение) на английском.

## Технический стек:
- Python 3.10+
- aiogram 3.x
- Для получения определений и примеров используй бесплатный Dictionary API (dictionaryapi.dev).
- Для получения перевода на русский используй бесплатный API перевода (например, MyMemory API).
- Обработка асинхронных запросов через `aiohttp`.

## Организация работы (ОЧЕНЬ ВАЖНО):
Перед написанием кода ты должен создать файл `PLAN.md`. В этот файл нужно записать пошаговый план разработки бота. 
По мере выполнения задачи ты должен обновлять этот файл, отмечая выполненные шаги (например, используя `[x]` вместо `[ ]`). Итоговое содержимое файла `PLAN.md` также должно быть предоставлено в ответе.

## Логика работы:
1. Пользователь отправляет сообщение.
2. Бот проверяет, что сообщение состоит только из букв (и пробелов для фраз).
3. Бот делает асинхронные запросы к выбранным API.
4. Если слово не найдено, бот сообщает об этом.
5. Бот форматирует ответ строго по шаблону и отправляет пользователю.

## Требования к форматированию ответа бота:
Ответ должен приходить обычным текстом (без Markdown-разметки вроде жирного шрифта или курсива, чтобы не мешать копированию). 
Текст должен быть структурирован так, чтобы пользователь мог легко скопировать нужные части.

Структура ответа (ровно две основные строки):
Строка 1: изначальное слово (пример использования на английском)
Строка 2: перевод (значение на английском)

Пример идеального ответа бота:
serendipity (Finding a beautiful old book in a dusty attic is pure serendipity.)
счастливая случайность (the occurrence and development of events by chance in a happy or beneficial way)

Пользователь должен иметь возможность легко скопировать определение, перевод или пример отдельно друг от друга.

## Требования к коду:
1. Раздели код на логические части (хэндлеры, функция запроса к API, форматирование). Можно сделать в одном файле `main.py`, но с четкой структурой.
2. Добавь комментарии к сложным местам.
3. Добавь обработку ошибок (таймауты API, ошибки сети, некорректный ввод).
4. Токен бота должен считываться из переменных окружения (используй `python-dotenv`).
5. Добавь команду `/start` с приветствием и инструкцией.

## Ожидаемый результат:
Предоставь в ответе следующее:
1. Содержимое файла `PLAN.md` с отмеченными выполненными шагами.
2. Полный код файла `main.py`.
3. Код для файла `.env` (с плейсхолдером для токена).
4. Список зависимостей для файла `requirements.txt`.
5. Подробную инструкцию по развертыванию и настройке бота (DEPLOYMENT.md). Инструкция должна включать:
   - Как получить токен через @BotFather.
   - Как настроить окружение локально (создание venv, установка зависимостей, настройка .env).
   - Как запустить бота локально.
   - Краткую инструкцию по деплою на недорогой Linux VPS (используя systemd или tmux/screen для постоянной работы).
